# Curavon Test Session Checklist v1

Use for every controlled test session.

---

## Before Session

- [ ] Fresh local account **or** delete all health data in Profile
- [ ] No real emergency scenario planned — use **fictional urgent examples only**
- [ ] Launch checklist reviewed (`docs/launch/launch-readiness-checklist-v1.md`)
- [ ] `npm run build` passed on facilitator build
- [ ] Test tasks ready (`docs/launch/real-user-test-script-v1.md` or subset)
- [ ] Feedback tracker open (`feedback-tracker-template-v1.md`)
- [ ] Consent script printed or on second screen (`tester-consent-script-v1.md`)
- [ ] Tester instructions shared (`tester-instructions-v1.md`)
- [ ] Facilitator read safety boundaries (`tester-safety-boundaries-v1.md`)

---

## During Session

- [ ] Consent script read; facilitator checkboxes complete
- [ ] Observe **without explaining too much** — let tester struggle briefly
- [ ] Record **confusion** (where, what task)
- [ ] Record **safety reactions** (scared, dismissed, trusted?)
- [ ] Record **hesitation points** (navigation, buttons, wording)
- [ ] Note if user believes Curavon is **diagnosing**
- [ ] Note if **one next action** is clear on Today / Ask / Guide result
- [ ] Capture **verbatim quotes** when possible
- [ ] Stop session if tester raises **real urgent concern** (see consent script)

---

## After Session

- [ ] Debrief questions asked (see feedback tracker)
- [ ] Top **3 issues** written
- [ ] **Severity** assigned (P0 / P1 / P2)
- [ ] **Fix / no-fix / defer** decided per `issue-triage-rules-v1.md`
- [ ] Feedback tracker updated
- [ ] Export or **reset local test data** if device shared
- [ ] P0 issues escalated immediately

---

## Facilitator Reminders

| Do | Don't |
|----|-------|
| Prompt tasks neutrally | Lead tester to “correct” answers |
| Record product behavior | Interpret medical symptoms |
| Use fictional scenarios | Ask for medication names or trauma details |
| Pause for real urgent concerns | Continue normal plan after urgent example fails |

---

## Quick Task Order (20–35 min)

1. Open app → onboarding/auth if fresh account  
2. Today → check-in  
3. Ask → normal concern (headache / stress / tiredness)  
4. Ask or Guide → **fictional** urgent (facilitator-led, not real)  
5. Guides → short flow  
6. Doctor Summary → view / generate  
7. Profile → export or delete (observe only; confirm if tester wants to try)

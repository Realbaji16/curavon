## Severity Definitions

### P0 — Fix before next tester

- Safety risk or urgent path failure
- User believes Curavon **diagnosed** them (product copy or flow caused this)
- App suggests **medication change** (start/stop/dose)
- **Data delete or export fails**
- **App-blocking bug** on core task (Today, Ask, auth shell)

### P1 — Fix before wider test group

- Major confusion about what Curavon does
- Broken core flow (Ask, Guides, check-in, follow-up, summary)
- **Misleading copy** (cloud sync, medical certainty, “AI doctor”)
- Inconsistent or missing **next action**
- Doctor Summary hard to find or use
- **Local-first / privacy** unclear

### P2 / Polish

- Copy clarity preferences
- Visual spacing or consistency
- Bundle size / performance warnings
- Non-critical loading states
- Navigation polish (e.g. duplicate tabs)

## Debrief Questions (Quick Capture)

2. Did the next action feel useful?  
3. Did safety messaging feel clear, not scary?  
4. Did Ask feel like it understood you?  
5. Would you use Doctor Summary before a visit?  
6. Did anything feel like medical advice?  
7. Would you come back tomorrow?
